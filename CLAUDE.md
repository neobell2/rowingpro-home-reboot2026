# Claude Code 행동 지침

## 1. 정보 처리 우선순위

**절대 원칙: 추론보다 사실 확인 우선**

### 우선순위
1. **직접 확인**: Read, Glob, Grep, git status, ls 등으로 사실 확인
2. **사용자 입력**: 제공된 정보 신뢰
3. **웹 검색**: 최신 정보 필요 시
4. **학습 데이터**: 일반적인 프로그래밍 지식

## 2. 금지 사항

### 절대 하지 말 것
- ❌ 파일을 읽지 않고 "있을 것 같다"고 추측
- ❌ git status 없이 커밋 상태 추론
- ❌ 사용자 승인 없이 커밋/푸시
- ❌ 학습 데이터만으로 최신 기술 정보 제공
- ❌ 이전 대화 내용만으로 현재 파일 상태 판단

### 대신 할 것
- ✅ Read, Glob, Grep 도구로 파일 직접 확인
- ✅ git status, git log로 실제 상태 파악
- ✅ 커밋 전 사용자에게 변경사항 보고 및 승인 요청
- ✅ 최신 정보는 웹 검색으로 검증
- ✅ 현재 상태를 직접 확인 후 작업

## 3. 작업 전 필수 확인

### 세션 시작 시 (필수)
1. `git log --oneline | head -5` - 최근 커밋 확인
2. `git status` - 현재 상태 확인
3. `PROJECT_BLUEPRINT_2026.md` 읽기 - 전체 설계 및 워크플로우 숙지
4. `SESSION_HANDOVER.md` 읽기 - 현재 진행 상황 및 다음 작업 파악
5. **문서 확인 전에는 코딩 시작 금지**

### 파일 수정 전
1. Read 도구로 파일 내용 확인
2. 파일 존재 여부 확인 (ls, Glob)
3. 수정 후 git diff로 변경사항 확인

### Git 작업 전
1. `git status` 먼저 실행
2. `git log --oneline | head -5`로 최근 커밋 확인
3. `git diff`로 변경사항 확인

### 코드 실행 전
- 의존성 설치 여부 확인
- 환경 변수 설정 확인
- 빌드 설정 확인

## 4. 프로젝트 컨텍스트

### 기술 스택
- **Framework**: Astro + Tailwind CSS
- **Runtime**: Node.js
- **Version Control**: Git + GitHub

### 참조 문서
- `PROJECT_BLUEPRINT_2026.md`: 프로젝트 설계 및 아키텍처
- `SESSION_HANDOVER.md`: 세션 간 인수인계 로그
- `CONTRIBUTING.md`: Git 워크플로우 및 커밋 규칙

### 작업 원칙
1. **승인 우선**: 모든 커밋은 사용자 승인 필수 (CONTRIBUTING.md 참조)
2. **문서 우선**: 불확실할 경우 참조 문서 먼저 확인
3. **현황 우선**: 과거 기록보다 현재 git status 우선

## 5. 응답 형식

### 작업 시작 시
```
[현재 상태 확인]
- git status 실행 결과
- 관련 파일 확인 결과

[작업 계획]
1. 단계별 작업 내용
2. 예상 변경 파일 목록

[승인 요청]
위 작업을 진행해도 될까요?
```

### 작업 완료 시
```
[완료 사항]
- 변경된 파일: path/to/file:line
- 주요 변경 내용

[다음 단계]
- 커밋 메시지 초안: "type: description"
- 커밋을 진행해도 될까요?
```

### 세션 종료 시
1. `SESSION_HANDOVER.md` 업데이트
2. 완료된 항목 [x] 처리
3. 다음 작업자를 위한 특이사항 기록
4. 사용자에게 최종 승인 및 커밋 요청
