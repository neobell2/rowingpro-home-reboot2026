## 결정 기록

### D-001: Claude Code를 사령탑(주력 도구)로 채택

```
결정: 여러 AI 중 Claude Code를 메인 작업자로 단독 채택

대안: Gemini(무료)를 메인으로 → 품질 우려로 폐기
변경 가능 여부: 낮음 (비용 구조 확정)
일시: 2026-02-02 (Session 5)

```
결정: 기존 Git 히스토리를 리셋하고 프로젝트를 새로 세팅

변경 가능 여부: 불가 (이미 실행됨)
일시: 2026-02-11 (Session 6)
```

### D-003: 스켈레톤보다 13년 업력 데이터 확보 우선

```
결정: Phase 2의 스켈레톤 구현을 미루고, 13년 전수 실적 데이터 발굴을 우선 수행
이유: "업력이 가장 강력한 마케팅 포인트"라는 디렉터 지시. 데이터 없이 껍데기만 만들면 로잉프로 정체성이 사라짐
대안: 스켈레톤 먼저 → 디렉터 판단에 의해 후순위로 변경
변경 가능 여부: 불가 (이미 실행됨, 418건 확보)
일시: 2026-02-12 (Session 8)
```

### D-004: 문서 가버넌스 시스템 전면 개편

```
결정: 파편화된 11개 시스템 문서를 CLAUDE.md(Hub) + 5개 핵심 문서로 재편
이유: 6개 규칙 파일 파편화, 핸드오버 비대화(359줄), 컬러값 충돌, DECISIONS 부재 등 구조적 결함으로 에이전트 실패 반복
대안: 기존 문서 유지하며 부분 수정 → Session 09에서 이미 같은 실패 발생, 구조적 해결 필요
변경 가능 여부: 낮음 (이미 실행 중)
일시: 2026-02-13 (현재 세션)
```

```

변경 가능 여부: 높음 (디벨롭 진행 예정)
일시: 2026-02-13 (현재 세션)
