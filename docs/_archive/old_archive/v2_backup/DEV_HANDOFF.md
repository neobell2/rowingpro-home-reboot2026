## 1. IA 구조 (확정)

### GNB 메뉴 (6개)

**영문:**
```
Home | About | Programs | Service | Track Record | Contact
```

**한글:**
```
홈 | 로잉프로 소개 | 대표 프로그램 | 서비스 | 운영 실적 | 문의하기
```

### 각 페이지 역할

**Home:**
- 정체성 한 문장
- 조정 = 구조 (혼자 불가 / 완주)
- 단일 CTA (문의)

**About:**
- 왜 조정인가 (구조 설명)
- 교육으로의 번역
- Hardware/People/Operation/Content 구조

**Programs:**
- 인도어 프로그램
- 아웃도어 프로그램
- 체험형 부스 프로그램
- MIX 프로그램

**Service (허브 + 드롭다운):**
- 클릭: 서비스 안내 허브 페이지
- 호버: 하위 5개로 직행
  1. 기업·조직 (HRD 적용/비적용 전면 배치)
  2. 학교·청소년
  3. 스포츠 이벤트·행사
  4. 클럽·일반
  5. 가족 체험 (비상품, 안내 전용)

**Track Record:**
- 운영 이력
- 운영 사례
- (네이밍 검토 중: "실적" 또는 "운영 이력")

**Contact:**
- 단일 진입
- 폼 내부 분기

## 4. 이미지 3슬롯 규칙 (확정)

### 비율 및 슬롯
- **비율:** 16:9 가로 (고정)
- **슬롯 3종:**
  1. `hero` - 히어로 이미지 (의식화/개회)
  2. `start` - 시작/과정 장면
  3. `summary` - 요약/종결 장면

### 동작 규칙

**모바일:**
- 자동 슬라이드 (3.5초 간격)
- 터치 시 정지
- 화면 이탈 시 정지

**데스크톱:**
- 자동 전환 금지
- 수동 네비게이션만 허용

### 파일명 컨벤션 (제안)
```
{card-name}-hero.jpg
{card-name}-start.jpg
{card-name}-summary.jpg

예:
employee-training-hero.jpg
employee-training-start.jpg
employee-training-summary.jpg
```

### 콘텐츠 원칙
- 실사진 우선
- 스톡/AI는 예외적 보조만
- 반드시 힘들고, 긴장되고, 전신을 쓰는 순간
- 웃음·체험 위주 컷 ❌
- 통제된 폭발 ⭕

### alt 텍스트 규칙
- 한글 1문장
- 장면 설명 (감정/평가 배제)

### 적용 범위
- **Service 하위 5개 페이지만** 사용
- Home, About, Programs, Track Record, Contact는 개별 디자인

## 5. PDF(화면 인쇄) 기능 (도입 보류)

### 설계 훅 예약
- 버튼 위치: 섹션 5 하단 (중·하단)
- 표시 문구: "요약 인쇄(PDF)"
- 출력 허용 섹션: 3, 4, 5 (요약 개요 / 프로그램 개요 / 구조·운영 요약)
- 출력 비허용: 히어로 / 이미지 / 강도 / FAQ / CTA

### 인쇄 CSS 준비
```css
@media print {
  /* 출력 허용 섹션 */
  .section-summary,
  .section-program,
  .section-structure {
    display: block;
  }
  
  /* 출력 비허용 */
  .hero,
  .image-slot,
  .section-intensity,
  .faq,
  .cta {
    display: none;
  }
  
  /* 워터마크 */
  @bottom-right {
    content: "출처: rowingpro.kr | " attr(data-page-name) " | " attr(data-date);
  }
}
```

### 구현 시점
- **즉시 구현 ❌**
- **결정 후 활성화 ⭕**
- 버튼·CSS는 코멘트 처리 후 배포

## 6. CTA 및 문의폼 (확정)

### CTA 전략
- **채널:** 문의폼 단일
- **패키지별 차이:** 없음
- **카드별 차이:** 없음 (숨김 필드로만 구분)

### 문의폼 구조

**입력 필드:**
1. 회사명 / 단체명 (필수)
2. 이름 (필수)
3. 직책 (필수) - 한국 B2B 필수
4. 연락처 (필수)
5. 이메일 (필수)
6. 문의 내용 (큰 텍스트 입력창)

**숨김 필드 (자동 전달):**
- `source_page`: 페이지 코드 (예: `employee-training`)
- `referrer`: 유입 경로
- `timestamp`: 제출 시각

**placeholder 문구 (문의 내용):**
```
원하시는 프로그램 유형, 인원, 일정, 예산 등을 자유롭게 적어주세요.
담당자가 확인 후 빠르게 연락드리겠습니다.
```

### 제출 후 동작
- 즉시 응답 화면 표시
- 이메일 자동 발송 (접수 확인)
- 내부 알림 (대표/담당자)

## 7. 이벤트 패키지 (확정)

### 패키지 네이밍
- **SIMPLE / CORE / PRO / MASTER**

### 패키지 성격
- 계약 조건 ❌
- 제공 한계 ❌
- 운영 상한 ❌
- **행사 규모·난이도·체급을 가늠하기 위한 기준점(anchor)** ⭕

### 가격 (시뮬레이션 안, CEO 결정 대기)
```
SIMPLE: 400,000원 (2시간 / 최소 구성)
CORE: 1,500,000원 (4시간 / 표준 운영)
PRO: 2,000,000원 (확장 운영)
MASTER: 3,000,000원 (총괄 포함)
※ 모두 VAT 별도
```

### 웹 표현
- 패키지명만 노출
- 가격·시간·인원은 **미정 상태 유지**
- 상세는 문의 후 제공

### 구현 지침
- 패키지 카드 UI 준비
- 가격·수치 입력 필드는 코멘트 처리
- 활성화 대기 상태로 배포

## 14. HRD 위치 정의 (중요)

### HRD는 Service 하위 항목이 아님
- Service 하위: 기업·조직, 학교·청소년, 이벤트, 클럽, 가족 (5개)
- HRD는 "기업·조직" 페이지 내부의 **설계 프레임(적용 모드)**

### 기업·조직 페이지 구조
```
기업·조직 페이지 (service--corporate)
  ↓
HRD 적용형 / 비HRD 적용형 분기 설명 (상단)
  ↓
공통 템플릿 (옵션 A) 진행
```

**설명:**
- 같은 기업도 니즈가 다름
  - HRD 적용: 신입사원 교육, 조직개발, 리더십
  - 비HRD 적용: 창립기념일, 직원가족 행사, 단합대회
- 로잉은 모든 HRD 세그먼트에 삽입 가능한 방법론적 모듈
- 웹에서는 HRD를 숨기지 않되 앞에 세우지 않음
