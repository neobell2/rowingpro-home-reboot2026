**일자:** 2026-02-03

---

## 1. 권한

| 역할 | 권한 |
|------|------|
| 사용자 | 최종 결정권자 |
| 담당 AI | 설계/판단/실행 (사용자 승인 하에) |
| WebGPT | 초안만 (검증 필수) |

---

## 2. 핵심 원칙

- 작업 1개 = 세션 1개
- 파일 1개씩 처리
- 전체 프로젝트 스캔 금지
- 추측 금지, 모르면 질문

---

## 3. 출력 규칙

**금지:** emoji, 인사, 감탄사, 불필요한 설명
**허용:** 실행 결과, 에러, 다음 단계

---

## 4. 파일 작업

- 생성/수정 전 경로 확인
- 전체 재작성 금지

- 완료 후 다음 파일

## 11. 모델 사용 전략 (Claude Code)

### 사용량 제한

| 모델 | 제한 |
|------|------|
| Sonnet | 주 40-80시간, 5시간당 45개 메시지 |
| Opus | 하루 30회 |
| Haiku | 무제한 (불필요) |

### 작업별 담당 및 모델 선택

| 작업 | 담당 | 모델 | 이유 |
|------|------|------|------|
| 프로젝트 셋업 | Claude | Sonnet | 초기 구조 중요 |
| 레이아웃 (헤더/푸터) | Claude | Sonnet | 복잡한 구조 |
| 템플릿 설계 | Claude | Sonnet | 재사용 설계 |
| Contact 폼 | Claude | Sonnet | 백엔드 판단 |
| Service 하위 5개 | Codex | - | 반복 구조 |
| Privacy, 404 | Codex | - | 단순 페이지 |
| Tailwind 스타일링 | Codex | - | 반복 작업 |
| 디버깅 (복잡) | Claude | Opus | 멀티 파일 추적 |
| 최종 검수 | Claude | Sonnet | 품질 관리 |

### 세션 프로토콜 (필수)

**시작 시:**
1. DASHBOARD.md 확인: 다음 작업
2. TEAM.md 참조: 담당 판단 (Claude/Codex)
3. 작업별 모델 선택 (상단 표 참조)
4. 사용자 알림:
   ```
   "이 작업([작업명])은 [담당]/[모델]로 진행합니다.
   예상 메시지: [N]개"
   ```
5. 사용자 승인 후 시작

**진행 중:**
- Sonnet 35/45 도달 시: 자동 경고
- Opus 25/30 도달 시: 사용자 확인
- 담당 변경 시: 사용자에게 알림

**종료 시:**
- 사용량 기록 (메시지 수/모델)
- DASHBOARD 업데이트

### 배치 원칙

- 파일 그룹화: 유사 작업 묶어 처리
- 단일 세션 = 단일 모델
- 메시지 횟수 최소화
