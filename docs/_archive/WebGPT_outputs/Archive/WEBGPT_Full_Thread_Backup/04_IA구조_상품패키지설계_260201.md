## 4. 상품 패키지 시뮬레이션

### 4.1 HRD Packages

#### START
- **대상**: HRD / 기관·단체
- **인원**: 10–20명 (최대 25명)
- **시간**: 2–3시간
- **구성**: 에르고미터 / 구조 이해 / 기본 동기화
- **인력**: 메인 코치 1, 운영 1
- **가격**: 300만–500만

#### CORE
- **대상**: HRD / 팀 단위
- **인원**: 20–40명
- **시간**: 반나절
- **구성**: 에르고미터 / 협업 미션 / 역할 분담
- **인력**: 헤드 1, 코치 1, 운영 1
- **가격**: 800만–1,200만

#### ADVANCE
- **대상**: HRD / 리더·핵심 인재
- **인원**: 12–24명
- **시간**: 1일
- **구성**: 에르고미터 + 보트 / 리더십 / 완주 설계
- **인력**: 헤드 1, 전문 코치 2, 안전·운영 1
- **가격**: 1,500만–2,500만

#### SIGNATURE
- **대상**: 고관여 조직
- **인원**: 8–16명
- **시간**: 1–2일
- **구성**: 보트 중심 / 고난도 동기화 / 완주
- **인력**: 디렉터 1, 헤드 1, 코치 2–3, 안전·운영 2
- **가격**: 3,000만~

## 6. 핵심 설계 원칙

### 6.1 Home 구조 시각화
- 카피만으로 전달 불가
- **정적 1컷 비교 시각화 + 짧은 문장**이 최적
- 종목 설명 ❌ / **구조 인식 ⭕**

### 6.2 가격/상세/FAQ 원칙
- 가격표 공개 ❌
- **가격 범위(Band) 표기 ⭕**
- **가격 산정 기준 설명 ⭕**
- 일정 고정 ❌

### 6.3 Contact 구조
- 상단 다중 분기 ❌
- **단일 진입 ⭕**
- **분기는 폼 내부에서 처리**
- "잘 모르겠습니다"는 정상 루트

### 6.4 상세 정보 제공 범위
- **웹**: 이해·판단 수준까지만
- **세부 실행**: 상담 이후
- **일정 고정** ❌
- **FAQ**: Contact 직전 또는 폼 내부

## 7. IA와 SEO 분리 설계

### 7.1 분리 이유

#### IA 기준
- **목적**: 신뢰, 이해, 문의 전환
- **대상**: HRD·기관·학교 담당자
- **결과**: 메뉴 수 적고, 구조 명확

#### SEO 기준
- **목적**: 검색 유입 확보
- **대상**: 아직 문제 정의가 안 된 잠재 고객
- **결과**: 키워드별 랜딩 다수 필요

**→ SEO를 IA에 그대로 올리면 메뉴가 깨짐**  
**→ IA를 SEO에 그대로 쓰면 검색 유입이 약함**

### 7.2 권장 구조

#### IA (네비게이션에 보이는 구조)
```
Home
About
Programs
Track Record
Contact
```
→ 여기에는 SEO 욕심을 넣지 않음

#### SEO 레이어 (네비게이션 비노출)
페이지는 존재하지만 메뉴에는 안 보이는 구조

**고객 × 니즈 키워드용:**
- `/hrd-teamwork-training`
- `/school-sports-program`
- `/youth-rowing-program`
- `/leadership-camp-rowing`
- `/sports-event-program`

**상품 × 방식 키워드용:**
- `/indoor-rowing-program`
- `/outdoor-rowing-program`
- `/rowing-experience-booth`
- `/mixed-rowing-program`

**장기 확장:**
- `/rowing-academy`
- `/rowing-coach-training`

→ 검색 유입 → 내부 링크 → Contact로 수렴

### 7.3 SEO 페이지 통제 규칙

**SEO 페이지에서도:**
- ❌ 가격 비교 프레임
- ❌ 가족 상품화
- ❌ 레저/체험업 뉘앙스

**항상 사용:**
- "조정 교육"
- "운영"
- "프로그램 문의"
