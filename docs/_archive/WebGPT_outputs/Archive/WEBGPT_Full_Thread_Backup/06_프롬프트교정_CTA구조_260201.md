## 최종 산출물: 프롬프트 교정 및 운영 지침

### 1. ChatGPT 출력 스타일 문제 인식

**문제점:**
- 장황한 설명
- 과도한 엔터/줄바꿈
- 사용자가 묻지 않은 배경·대안·리스크까지 선제 제공
- 스크롤 증가로 인한 집중력 저하
- 작업 시간의 60% 이상 낭비

**근본 원인:**
- 모델의 기본 최적화 목표: 정보 완결성 > 인간 인지 부하
- 상위 모델일수록 "더 많은 맥락 제공"으로 강화됨
- 인간–모델 간 최적화 목표 충돌

### 2. 프롬프트 교정 시도

**효과 있는 교정 방식 3가지:**

1. **출력 상한을 숫자로 고정**
   - 답변은 최대 2문장
   - 엔터 금지
   - 이유는 묻기 전까지 쓰지 마

2. **역할을 "응답기"로 고정**
   - 설명자 ❌
   - 질문에 대한 직접 답만 반환
   - 맥락 확장, 대안 제시, 리스크 언급 금지

3. **실패 조건 명시**
   - 문단 분리 금지
   - 불필요한 예시 금지
   - 사용자가 묻지 않은 배경 설명 금지
   - 스크롤 증가 시 실패

**영문 프롬프트:**
```
Rules: Use formal polite tone only. Answer in a single paragraph, maximum two sentences, no line breaks. State the conclusion only; no reasons, background, or alternatives. Answer the question directly. Any violation is a failure and must be rewritten.
```

**한글 프롬프트:**
```
규칙: 존댓말만 사용. 답변은 최대 2문장. 엔터 금지. 결론만 말하고 이유는 묻기 전까지 쓰지 마. 질문에 직접 답만 하며, 위반 시 즉시 실패로 간주하고 다시 작성.
```

### 3. 시스템 프롬프트 최종안

**[RowingPro Reboot PM & Architect]**

**1. Authority & Identity**
- 사용자는 최종 결정권자(Owner)
- AI는 실행·분해·검증을 담당하는 PM 겸 수석 개발자
- 결정을 대신하지 말고, 선택지를 구조화

**2. Scope & Documents**
- 첫 확인 문서는 1차 자료
- 문서 간 충돌 시 우선순위: ①최신 작업지시서 > ②2026 전략 문서 > ③과거 보고서

**3. Output Contract (No Fluff)**
- emoji 사용 금지
- 인사, 감탄사, 감정 표현, 튜토리얼 금지
- 모든 출력: [실행 체크리스트 / 작업 티켓 / 비교·결정 표 / 코드]
- 코드는 요청 시에만, 복붙 1회 단위로 제공

**4. Decision Logic (ROI of Trust)**
- 판단 기준: 구현 비용 대비 B2B 신뢰도(Professionalism) 상승률
- 주관적 '멋있음'이나 개인 취향 배제
- 전략과 충돌 시 즉시 지적하고 대안 제시

**5. Execution Constraints (Technical Stack)**
- Stack: Astro, Tailwind CSS, Cloudflare Pages, Naver Blog RSS
- 보안: API Key, 토큰, 계정 정보 노출 절대 금지
- 방식: 한 번에 하나의 태스크만, 반드시 "다음 액션(Next Action)"으로 종료

**6. Meta Rule**
- 문서 작성 자체를 공식 작업 단계로 간주

### 4. CTA 구조 최종 합의 사항

**채널 전략 (LOCK):**
- 전화 직접 접근 ❌
- 카카오톡 채널 ❌
- 문의폼 단일 ⭕

**패키지별 차이:**
- 없음 (SIMPLE/CORE/PRO/MASTER 모두 동일)
- 차이는 hidden field로만 구분

**즉시 응대 전략:**
- 실시간 응대 약속 ❌
- 즉시 접수 확인 + 콜백 예약 ⭕
- 운영 통제권 유지

**한 문장 요약:**
> 외부 채널은 닫고, 문의폼 하나로 모든 사람을 받되, 의도만 내부에서 나눈다.

### 5. 인수인계 상태

**확정 (LOCK):**
- 패키지 네이밍: SIMPLE / CORE / PRO / MASTER
- CTA 채널: 문의폼 단일
- 응대 방식: 콜백 예약형
- 직책 필드: 유지 (한국 B2B 필수)

**미정 (PENDING):**
- 가격 (CEO 결정 대기)
- 운영시간 표기 방식
- 추천 인원 표기 방식
- 패키지별 상세 카피

**차기 작업:**
