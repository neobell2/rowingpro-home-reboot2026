## 1. BI 프레임워크 선정

### 최종 선정 프레임 (3개)

#### ✅ Brand Positioning Statement (핵심)
**이유:**
- 홈페이지의 1차 목적이 B2B 신뢰 확보 + 문의 전환
- 한 문장으로 "왜 우리인가"를 고정 가능

**적용 구조:**
```
로잉프로는
[기업 HRD 담당자]를 위해
[국가대표 출신이 직접 설계한]
[검증된 팀빌딩/조직개발 프로그램]을 제공하는
[전문 교육 파트너]다.
```

#### ✅ Brand Pyramid (Unilever)
**이유:**
- 감성보다 논리·증빙·구조에 강함
- 홈페이지 섹션 구조 설계와 1:1 매칭 가능

**적용 매핑:**
| Pyramid 단계 | 로잉프로 대응 |
|-------------|--------------|
| Attributes | 국가대표 출신, 13년 업력, 협회 주관 |
| Functional Benefits | 검증된 팀워크 형성 |
| Emotional Benefits | "실패하지 않는 선택" |
| Values | 전문성, 책임, 실전성 |
| Essence | 국가대표가 설계한 팀빌딩 |

#### ✅ STP (Segmentation-Targeting-Positioning)
**이유:**
- 현재 프로젝트의 핵심 문제: 누구에게 말하느냐
- B2C / B2B / HRD / 워크숍 혼선 정리 필요

**적용 결과:**
- Segmentation: 기업 HRD / 공공기관 / 일반 체험
- Targeting: 기업 HRD 담당자
- Positioning: 체험형 레저 ❌ → 전문 교육 솔루션 ⭕

### 검토 후 제외된 프레임

#### Brand Key (Unilever)
- 글로벌 대기업의 BI 통제용 마스터 키
- 다수의 실행 주체(마케팅, 디자인, 대행사) 전제
- **판단**: 단일 브랜드·1인 중심 조직에는 과도한 문서화

#### Golden Circle (Simon Sinek)
- 메시지 압축에는 유효
- 단독 BI 프레임으로는 불충분
- **판단**: Hero 문구 / About 서두에 한정 사용 (보조 도구)

#### Brand Identity Prism (Kapferer)
- 유럽 대기업·럭셔리 브랜드용
- 감성·이미지 축 비중 큼
- **판단**: Culture / Relationship 일부만 체크리스트로 차용

#### IDEO Strategic Brand Foundation
- 혁신조직·디자인 조직용
- 워크숍 전제, 다기능 조직에 효과적
- **판단**: 현재 프로젝트 스케일과 불일치

## 2. 작업 순서 및 프레임 위계

### 프레임 간 위계 구조
```
[1] STP
   ↓
[2] Brand Positioning Statement  ← 절대 기준
   ↓
[3] Brand Pyramid
   ↓
[4] Golden Circle (Why/How 보조)
   ↓
[5] Identity Prism (검증용)
```

**중요**: Positioning이 틀리면 나머지는 전부 무효

### 실제 작업 순서

#### STEP 1. STP 고정 (판단 단계)
- Segmentation: B2B / B2C / 공공
- Targeting: 기업 HRD 담당자
- Positioning 방향: 체험 ❌ → 전문 교육 ⭕

#### STEP 2. Brand Positioning Statement 1문장 확정 (결정 단계)
- "누구에게 / 무엇을 / 왜 우리인가"
- 홈페이지, 블로그, 제안서 모두 동일 문장 사용
- **이 문장이 BI의 실질적 코어**

#### STEP 3. Brand Pyramid 작성 (구조 단계)
- 주장은 무엇인가
- 근거는 무엇인가
- 증빙은 무엇인가
- 결과 → 홈페이지 섹션 구성 (Home → About → Programs → Track Record)

#### STEP 4. Golden Circle 축약 적용 (표현 단계)
- Why: 시장 내 존재 이유 (철학 ❌)
- How: 국가대표 직접 설계
- What: 기업 팀빌딩 프로그램
- 용도: Hero 문구 / About 첫 문단 전용

#### STEP 5. Identity Prism으로 최종 점검 (검수 단계)
- 톤이 과장됐는가?
- 관계 설정이 애매한가?
- 감성 언어가 전문성을 침식하는가?
- **수정은 표현만, 포지션은 건드리지 않음**

## 3. HRD 시장 기능 분석 (핵심 통찰)

### 로잉프로의 실제 시장 기능

**정의:**
> 로잉프로는 HR 담당자가 위로부터도, 옆으로부터도 욕먹지 않게 만들어주는 동시에,  
> 조직에는 "힘들었지만 납득 가능한 경험"을 남기는 안전한 집단 훈련 장치다.

### HR 담당자가 겪는 4가지 리스크

#### 리스크 1. 위로부터의 평가
- "비용 대비 효과 있었나?"
- "팀워크 교육이 실제로 도움 됐나?"
- → 정량 지표 없으면 부정적 평가

#### 리스크 2. 옆으로부터의 평가
- "너무 쉬우면 예산 낭비"
- "너무 힘들면 불만 폭주"
- → 강도 조절 실패 시 양쪽에서 공격

#### 리스크 3. 사고 책임
- 외부 활동 특성상 사고 가능성
- 사고 발생 시 담당자 책임
- → 안전 통제 불가능하면 선택 불가

#### 리스크 4. 참가자 불만
- "억지로 끌려왔다"
- "시간 낭비다"
- "왜 이걸 하는지 모르겠다"
- → 참가자 반발 시 프로그램 무효

### 로잉프로의 리스크 대응 구조

#### 기능 1. 상부 보고용 정당성 제공
- "국가대표 출신" = 전문성 증빙
- "한강 미사리" = 공신력 증빙
- "13년 업력" = 검증 완료 시그널
- → 상부에 "제대로 알아보고 선택했다" 근거 제공

#### 기능 2. 현장 통제 가능성
- 조정 = 구조적으로 통제된 운동
- 개인 경쟁 ❌
- 과속 ❌
- 돌발 행동 ❌
- → "조직 훈련 제대로 시켰다"로 해석 가능

#### 기능 3. 참가자 납득 구조
- 모두 동시에 힘듦
- 개인만 고통받지 않음
- 빠질 수 없는 구조
- → "나만 손해 봤다" ❌ / "다 같이 했다" ⭕

#### 기능 4. 사고 가능성 최소화
- 집단 동기화 운동
- 변수 적은 통제 환경
- 전문 지도자 상시 동행
- → 안전한 집단 훈련 장치

### 한 줄 정의
> **로잉프로의 진짜 기능은 '팀워크 교육'이 아니라**  
> **'구매자의 리스크를 대신 관리해주는 조직 경험 설계'입니다.**

## 4. 코로나 이후 시장 변화 대응

### 시장 변화 현황

#### HRD 시장 축소
- 대규모 공채 없음
- 집합형 교육 축소
- 기업 HRD 예산 감소

#### 비기업 시장 확대
- 개인, 소규모 집단
- 동호인, 학교
- 유학생 시장
- **가족 고객 급증** (코로나 이후)

### 문제 인식

#### 현재 상황
- 시장 단위는 달라졌는데
- 우리가 하나의 '최종어(팀워크)'만을 들고 있는 상태

#### 시장별 언어 마찰
- HRD 시장: 팀워크 = 구매 언어 ⭕
- 비기업 시장: 팀워크 = 구매 언어 ❌ (부담·오해 유발)

### 비기업 시장의 실제 구매 키워드

#### 공통 욕구
- 혼자서는 하기 어려운 경험
- 처음이지만 진입 가능한 구조
- 몸으로 느끼는 몰입
- 낯선 사람과도 안전하게 연결되는 경험
- "나도 해냈다"는 자기 효능감
- 관계 강요 없음, 성과 평가 없음

**이 시장에서 팀워크는:**
- 목표 ❌
- 결과 ❌
- 주장 ❌
- **부수 효과 ⭕**

### 상위 개념 도출 (핵심)

#### 로잉프로의 실제 코어
> **"혼자서는 성립되지 않는 경험을,**  
> **안전하게 완주하게 만드는 구조"**

#### 시장별 번역 구조
| 시장 | 번역 |
|-----|------|
| HRD | 팀워크 |
| 개인 | 연결감 / 몰입 / 성취 |
| 학교 | 협력 학습 |
| 동호인 | 공동 몰입 |
| 유학생 | 관계 진입 장치 |
| 가족 | 자연스러운 함께함 |

**중요**: 팀워크를 버리는 것 ❌ / 팀워크를 포함한 상위 개념 사용 ⭕

## 5. 가족 고객 전략 (최종 합의)

### 현황 인식 (팩트)

#### 상황
- 코로나 이후 가족 고객 **실제 증가**
- 일시적 실험 단계 아님
- 대응 지연 시 기회 손실 발생

#### 가족 고객의 본질적 위치
- 신규 확장 ❌
- 사이드 프로젝트 ❌
- **로잉프로 본질이 가장 자연스럽게 작동하는 사용자 타입 ⭕**

#### 가족이 특별한 이유
가족은:
- 강요된 팀워크 없음
- 평가 없음
- 교육 목적 없음

그럼에도 불구하고:
- 자연스럽게 함께하게 됨
- 리듬·속도·완주가 필연적으로 발생

### 웹 노출 원칙 (합의 완료)

#### 절대 금지
- 가족 상품 카드 노출 ❌
- 가격표 노출 ❌
- 예약/구매 CTA ❌

#### 반드시 충족해야 할 조건
1. **우리는 가족 대상 운영을 한다는 사실은 명확히 노출**
2. 가족을 '상품'이 아닌 **작동 사례/활용 유형**으로 제시
3. 흥미가 생긴 잠재 고객에게는 **반드시 다음 이동 경로 제공**

### 웹 구조 적용 방향

#### 상품 섹션
- 기업 / HRD / 인도어 / 아웃도어
- 가격, 구성, 운영 방식 명시
- **가족 항목 제외**

#### 비상품 섹션 (신설 또는 강화)
**섹션명 예:**
- "Who This Works For"
- "이 경험이 작동하는 사람들"

**포함 항목:**
- 기업 조직
- 학교·교육기관
- **가족**
- 개인/동호인

#### 가족 항목 표현 원칙
- 상품명 ❌
- 가격 ❌
- 일정 ❌
- 짧은 설명 + 사진
- **'이것도 된다'는 인지 수준까지만 허용**

### 가족 전용 CTA 처리

#### 버튼 문구 예시
- "가족 프로그램은 별도로 안내드립니다"
- "우리 가족에게 맞는 방식이 궁금하다면"

#### 연결 경로
- 가족/개인 문의 전용 폼
- 또는 문의 단계에서 '가족' 체크 분기

#### 목표
- 호기심 단계에서 필터링
- 진성 리드만 상담 파이프라인 유입
