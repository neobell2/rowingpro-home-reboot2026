## 확정된 핵심 사항

### 1. 정체성 (Identity)

#### 정체성 한 문장
```
로잉프로는
조정을 통해
개인의 한계를 노출시키고,
팀 안에서 그것을 책임지게 만드는
교육 집단이다.
```

#### 핵심 키워드
- 인내
- 극기
- 희생
- 헌신
- 팀워크

#### 중요한 인식
- ❌ "고급화 전략" 조직 아님
- ❌ "엘리트 마케팅" 조직 아님
- ✅ "누구에게나 열려 있지만 아무나 통과할 수 없는 구조"
- ✅ 노블리스 = 특권이 아니라 자기 규율

### 2. 전환 정책 (Conversion)

#### 공식 전환 수단 (3가지)
1. **문의 폼** (1순위)
2. **대표전화** (긴급 0순위)
3. **대표이메일** (비동기 보조)

#### 백업 수단
- **카카오톡 채널**: 백업으로 인정, 당장 개설 안 함
- **텔레그램**: 보류, 향후 채택/드롭 반드시 결정

#### 스마트스토어
- **운영 리드** (Operational Lead)로 분류
- 영업 리드와 분리
- Self-serve 구매 퍼널
- 홈페이지 전환 설계와 분리

### 3. 리드 분류

#### 영업 리드 (Sales Lead)
- 출처: 홈페이지 문의폼, 전화, 이메일
- 목적: 제안, 조율, 확장
- B2B 파이프라인

#### 운영 리드 (Operational Lead)
- 출처: 스마트스토어 결제, Q&A
- 목적: 일정, 옵션, 서비스 제공
- CS/운영 트랙
- B2B 확장 신호 시 영업 리드로 승격

### 4. 영문판 정책

#### 기본 구조
- **K-K 팩 + E 팩** (국문 기준, 영문은 번역 파생)
- 구조 완전 동일
- Contact 섹션도 동일

#### 영문판 범위
- Home 요약
- About 요약
- Programs 개요
- Track Record 요약
- Contact (동일)

#### 영문판 특징
- 영문 SEO 안 함
- 번역은 GPT
- 목적: 외국인 임원 참고용 (영업 아님)
- 비용: 초기 0원 (GPT 번역), 유지 연 1-2시간

#### Contact 표기
```
Phone: +82-... (Korean language only)
```

### 5. 법적/신뢰 고지 (Legal Baseline)

#### 푸터 필수 표기
- 상호
- 대표자
- 주소 (텍스트만, 지도 링크 없음)
- 전화 + 이메일
- 사업자등록번호

#### 개인정보 문구
- 간이 문구 1줄
- Contact 페이지 하단만
- 보관/파기 범위만 표기 (구체 일수 미기재)
- 전용 페이지 미생성

### 6. 브로셔 정책

#### 브로셔 정의
- **1~2페이지 보고용 요약 자료**
- 목적: 기업 내부 보고 보조
- 대상: HRD 실무자 → 상급자
- 성격: 설명용, 요약용 (설득/마케팅 아님)

#### 브로셔 vs 풀스펙
- 홈페이지 = 풀스펙 회사소개서
- 홈페이지 = 풀스펙 상품안내서
- 별도 풀스펙 PDF ❌
- 요청 시에만 개별 제출 ✅

#### 배포 방식
- 존재만 허용
- 배포 여부/방식은 IA 설계 이후 결정 (유보)

### 7. 조정의 이중 맥락

#### 한국 맥락
- 비인기, 비대중 종목
- 접근 장벽 매우 높음
- 시설 희소 (미사리 중심)
- **정보가 거의 없음 → 맥락 설명 필수**

#### 글로벌 맥락
- 엘리트 스포츠
- 문화 자본
- 아이비리그/옥스브리지 0티어
- 글로벌 기업 채용 시그널
- 럭비, 조정 = 0티어

### 8. HRD 시장 변화

#### 2013 이전: 극기 시장
- 산, 해병대, 혹한기
- 사고 다발 → 퇴출

#### 2010s 중반: 팀워크 시장
- 조정 = HRD 개념 최초 도입 (로잉프로)
- 카피캣 난립
- 시장 희석

#### 현재: 집체 HRD 붕괴
- 대규모 공채 없음
- 집합형 교육 축소
- **조정만 살아남음** (원래 자리로 복귀)

### 9. 고객군 증거

#### 2013-2015
- 대기업 HRD
- 삼성, 금융, 제조, 공기업
- 엘리트 실무/관리층

#### 2024
- 국제학교 학생 (아이비리그 준비)
- 부모: 외국 유학, 외국계 기업 종사자
- 외국인 임원이 먼저 권하는 경우 흔함

### 10. 기존 사이트 분석 결과

#### 문제점
- 웹브로셔 이상의 기능 못함
- 회사소개서 1.5를 HTML로 분해한 흔적
- 정보 과다/분산/중복
- 영업 동선 없음
- 사용자 구분 없음
- 보고용으로 쓸 수 없음

#### 급조 배경
- 자금 부족
- 시간 압박
- 외주 통제 가능한 유일 인력
- "일단 만들고 수정" → 코로나로 방치

## 미결정 사항 (다음 스레드에서 해결)

### 🔴 우선순위 1: 웹의 1차 목적 선택
```
A. 내부 보고용 자료 생성 (캡처/링크 전달)
B. 문의 전 '검증 단계' 통과
C. 상담 전 사전 이해 정렬
D. 위 세 개 모두
```

### 🟡 우선순위 2: 레이어 구조
"조정 설명 영역"과 "로잉프로 설명 영역"을
- 같은 레이어에 둘까?
- 분리할까?

### 🟡 우선순위 3: 선별 vs 통과
로잉프로는 사람을
- 선별하려는 조직인가?
- 통과시키려는 조직인가?

→ **답: 통과** (확정됨)
