## 최종 산출물: Contact 페이지 설계 완료 상태

### 1. Contact 페이지 구조 (LOCK)

**큰 제목**: 문의하기

**소제목**: 없음

**설명**: 아래 양식에 원하시는 서비스 내용과 정보를 남겨주시면 연락드리겠습니다.

**접수 완료 메시지**: 문의가 정상적으로 접수되었습니다.

### 2. 문의폼 필드 스펙 (LOCK)

**필수 필드:**
- 성함
- 회사
- 이메일
- 연락처

**선택 필드:**
- 부서 (선택)
- 직함/직급 (선택)
- 홈페이지 (선택)

**적용 원칙:**
- placeholder·설명 문구 미사용
- 국문 라벨 통일
- (선택) 표기 명시
- 선택 항목 미입력 시에도 제출 가능

### 3. 전화 노출 규칙 (LOCK)

**노출 방식**: 정보 레벨 텍스트 노출(버튼·링크·아이콘 없음)

**하단 문구**: Korean language only · Available 09:00–18:00 (UTC+9)

**위치**: Contact 하단 또는 Footer(주 CTA 영역과 분리)

**의미**: 결심 고객 직통 경로, CTA 아님

### 4. 개인정보 동의 형식 (LOCK)

**형식**: 체크박스 선행 + 문장 후행

**문구**: 
```
☐ 본인은 개인정보 처리방침 및 개인정보 수집·이용에 동의합니다.
```

**링크**: 단일 개인정보 처리방침 페이지 연결

**필수 여부**: ⭕ (미체크 시 제출 불가)

### 5. 유입 맥락 기록 설계 (LOCK)

**설계 용어**: page_id (페이지 식별자)

**저장 컬럼**: source_page

**원칙**: 
- 모든 페이지에 고유 page_id 부여
- 모든 문의 진입 시 source_page로 기록
- 중복·누락 방지: 페이지 단위 1키, 폴백값 unknown

**구현 방식**:
- 모든 문의 링크: ?source={page_id}
- Contact 폼: hidden field로 저장
- 사용자 노출: 없음

**효과**:
- 단기: 응대가 빨라지고 정확해짐
- 중기: 고칠 곳이 명확해짐
- 장기: 전략 판단의 데이터 자산이 됨

### 6. 문의/응대 채널 구조 (LOCK)

**채택 범주**: A (비동기·기록형 채널)

**배제**: 
- 전화 직접 접근 ❌
- 카카오톡 실시간 ❌
- 이메일 직접 ❌

**채널**: 문의폼 단일

**패키지별 차이**: 없음 (SIMPLE/CORE/PRO/MASTER 모두 동일)

**운영자 인지**: 
- 이메일 또는 Google Chat 알림
- 구체 수단: 결정 보류 (구현 시 확정)

### 7. ChatGPT 출력 스타일 교정

**문제점 인식**:
- 장황한 설명, 과도한 엔터/줄바꿈
- 사용자가 묻지 않은 배경·대안·리스크까지 선제 제공
- 작업 시간의 60% 이상 낭비

**시스템 프롬프트 수정안 (압축형)**:
```
출력 형식은 목적 기반으로 선택한다.
결정 고정·진행 관리·인수인계 목적일 경우 체크리스트/티켓/표를 기본으로 사용한다.
개념 설명·정합성 논의·기억 정렬·방향 조율 목적일 경우 서술형을 우선하며 불렛 사용을 강제하지 않는다.
사용자가 표현 방식에 혼란·피로·불만을 보일 경우 즉시 형식을 완화하고 서술형으로 전환한다.
```
