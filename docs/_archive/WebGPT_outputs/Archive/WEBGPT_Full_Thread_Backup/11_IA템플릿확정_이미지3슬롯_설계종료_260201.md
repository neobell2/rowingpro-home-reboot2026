### 2. 이미지 3슬롯 규칙 (확정)

#### 비율 및 슬롯 정의

**비율:**
- 16:9 가로 (고정)

**3슬롯 구성:**
1. **hero** - 히어로 이미지 (의식화/개회)
2. **start** - 시작/과정 장면
3. **summary** - 요약/종결 장면

#### 동작 규칙

**모바일:**
- 자동 슬라이드 (3.5초 간격)
- 터치 시 정지
- 화면 이탈 시 정지

**데스크톱:**
- 자동 전환 금지
- 수동 네비게이션만 허용

#### 콘텐츠 원칙

**이미지 선정:**
- 실사진 우선
- 스톡/AI는 예외적 보조로만 허용

**장면 기준:**
- 반드시 힘들고, 긴장되고, 전신을 쓰는 순간
- 웃음·체험 위주 컷 ❌
- 통제된 폭발 ⭕

### 3. 템플릿 옵션 최종 선택

**검토한 옵션:**
- **옵션 A**: 히어로 → 이미지 3슬롯 → 요약 → CTA
- **옵션 B**: 히어로 → 요약 → 이미지 3슬롯 → CTA
- **옵션 C**: 히어로 → 이미지 1장 → 요약 → 이미지 2장 → CTA

**최종 선택: 옵션 A**

**이유:**
- 막내 필터 통과 (상단 안전 신호)
- 장면 몰입 (이미지 3슬롯 연속)
- 텍스트 부담 최소

### 4. 유지보수 체크리스트

#### 이미지 교체

**판단 기준:**
- 3슬롯 규칙(hero/start/summary) 유지 → 단순 교체 → 운영자 처리
- 슬롯 수·순서·비율 변경 → 구조 변경 → 대표 승인

**운영자 권한:**
- 슬롯 내부 이미지 변경만 가능
- 규칙 변경 불가

#### PDF(화면 인쇄)

**판단 기준:**
- 도입 여부 결정 → 대표 승인
- 버튼 문구·위치 조정 → 운영자 처리

#### IA/명칭

**판단 기준:**
- 카드 명칭 변경 → 대표 승인
- 설명 문구 수정 → 운영자 처리

**변경 판단 기준 요약:**
- **단순 교체/추가**: 운영자 단독 처리
- **구조·정책 변경**: 대표 승인 필수

### 5. 크롬 페이지 캡처 관련 검토

#### 현실 인식

**크롬 확장프로그램:**
- 페이지 전체를 이미지로 저장 가능
- 예: GoFullPage, Full Page Screen Capture, FireShot

**웹 대응 가능성:**
- 완전 차단 불가능
- 우클릭 방지, 선택 방지는 우회 가능 → 실효성 낮음

#### 대응 전략

**기본 전제:**
- 유통은 허용 전제, **범위만 통제**

**콘텐츠 층위 분리:**
- 출력 가치 있는 정보 → 요약 섹션에만 배치
- 강도·밀도·상세 → 비출력 영역 유지

**결론:**
- PDF 인쇄 정책과 동일한 전략
- 현재 설계 방향 정합적

### 6. PDF(화면 인쇄) 정책 확정 요약

#### 목적

자료 제공이 아니라 **보고서화 가능성에 대한 암묵적 승인 신호**

#### 형태

브라우저 인쇄(Print to PDF) 트리거

#### 전제

파일 저장 차단 불가 → **출력 범위만 통제**

#### UX 위치

- 상단 노출 금지
- 카드 랜딩 내부 **중·하단** 배치
- CTA(문의)와 직접 경쟁 금지

#### 출력 범위

**허용:**
- 요약 개요
- 프로그램 개요
- 프로그램 구성 요약

**비허용:**
- 강도·상세 과정
- 현장 밀도 높은 이미지

#### 이미지 포함 정책

**고정 규칙 없음** (카드·시점별 선택 가능)

#### 품질 관리

- 워터마크/푸터 고정: 출처·페이지명·날짜
- 맥락 절단 방지
- 저장·전달 가능성 **허용 전제**

#### 현재 상태

**도입 여부:** 보류  
**설계 훅:** 예약 완료
