# 로잉프로 랜딩페이지 구조 및 PDF 인쇄 기능 검토

**날짜**: 2026년 2월 1일  
**세션 유형**: ChatGPT 4 세션  
**주요 주제**: '왜 조정인가' 실행 기준, 랜딩페이지 구조, PDF 인쇄 기능 정책

---

## 최종 산출물: 확정 사항 및 정책 문서

### 1. 확정된 IA/구조

**전체 구조:**
- Service → 기업/기관/단체 → 목적 기준 5카드 → 카드별 개별 랜딩페이지

**카드 5종 (고정):**
1. 사원교육 / 조직개발
2. 팀빌딩 / 협업
3. 리더십
4. 행사 / 체육대회
5. 건강 / 웰니스 (건강증진프로그램)

**카드별 랜딩페이지:**
- 공통 템플릿 사용
- 콘텐츠(텍스트·이미지)는 카드별 개별
- 개별 URL·개별 페이지

**이미지 원칙:**
- 모든 랜딩페이지 **이미지 3장 통일**
- 히어로 1 + 운영/과정 1 + 결과/분위기 1
- 실사진 우선, 필요 시 스톡/AI는 예외적 보조로 허용

**CTA:**
- 전 페이지 **'문의하기'로 통일**
- 글로벌 문의폼과 동일
- 카드/페이지 맥락은 **숨김 필드로 자동 전달**

**문의폼:**
- 입력 항목 추가 없음
- 하단 큰 텍스트 입력창에 안내 문구(placeholder)로 정보 유도

### 2. '왜 조정인가' 실행 기준점

**본질 기준:**
- 조정은 극기·계발·건강이 아니라 **목적을 가진 놀이 구조**로 다룬다
- 즐거움·유희·쾌락은 삭제 대상이 아니라 **보존 대상**이다
- 완결을 향한 밀어넣기가 사라지면, 그 순간 조정은 다른 무엇이 된다

**인간 기준:**
- 페르소나는 직무·연령·성별이 아니라 **놀이에 진입한 상태**로 판단한다
- 혼자 완결될 수 없다는 전제를 숨기지 않는다
- 개인의 쾌락은 집단의 완결에 종속되는 구조를 약화시키지 않는다

**표현 기준:**
- 설명은 최소, 장면이 우선이다
- 말로 납득시키려는 순간, 잘못된 방향이다
- 이미지는 **"보는 즉시 이해되는가"**만 통과 기준이다

**웹/구조 기준:**
- B2B 문법은 전제가 아니라 **결과적으로 읽히는 효과**여야 한다
- CTA는 단일 진입만 허용한다 (놀이의 문은 하나)
- 톤은 격식도 감성도 아닌 **훈련을 다루는 사람의 태도**를 유지한다

**제거 기준:**
- 설명 ❌
- 비장함 ❌
- 이벤트성 ❌
- 격식주의 ❌

---

### 3. 톤앤매너 / 콘텐츠 원칙

**즐거움:**
- 스포츠의 본질이며 **시작·과정·결과 전 구간에 존재**
- 가벼운 재미가 아니라 완결을 향해 몸을 던질 때 발생하는 **깊고 정직한 즐거움**

**표현 방식:**
- 비장함·이벤트성 배제
- **관리되고 통제된 스포츠**
- 설명 최소, 장면 중심

**이미지 흐름:**
- 의식화(개회) → 구조(동기화) → 종결
- 3장 고정 구조

---

### 4. 페르소나 / 노출 순서

**1차 필터: '막내'**
- 상단에서 안전·정식·관리 가능성만 확인
- "무리 없는지 / 사고 안 나는지 / 설명 가능한지"

**강도 노출:**
- **선택 진입 이후** 노출
- 상단은 통과 신호 중심

**C레벨 인식:**
- **"빡세지만 괴롭지 않음"**을 동시에 인식
- 놀이 구조는 유지하되, 관리되는 스포츠

### 5. PDF로 인쇄하기 기능 (정책 문서화 완료 / 도입 보류)

#### 기능 정의

**형태:**
- 브라우저 인쇄(Print to PDF) 트리거

**목적:**
- 자료 제공이 아니라 **보고서화 가능성에 대한 암묵적 승인 신호**

**전제:**
- 파일 저장 차단 불가 → **출력 범위만 통제**

#### UX 위치 원칙

**배치:**
- 상단 노출 금지
- 카드 랜딩 내부 **중·하단** 또는 **요약/개요 섹션 하단**에만 배치
- CTA(문의)와 직접 경쟁 금지

#### 출력 범위 정책

**허용:**
- 카드 요약
- 프로그램 개요
- 운영 구조(요약)

**비허용:**
- 강도·상세 과정
- 현장 밀도 높은 이미지

**방식:**
- 인쇄 전용 CSS로 **섹션 단위 출력**

#### 품질·리스크 관리

**워터마크/푸터 고정:**
- 출처·페이지명·날짜

**맥락 절단 방지:**
- 요약 문장 유지
- 장면 과잉 배제

**유통 전제:**
- 저장·전달 가능성 **허용 전제**로 설계

#### 운영 정책

**명확한 구분:**
- 브로셔/회사소개서 대체 ❌
- 문의 우회 수단 ❌
- 대표·A 추가 대응 없음 (자동 기능)

#### 득실 판단 요약

**득:**
- 보고서 문화 정합 신호 제공
- 막내/과장 단계 리스크 회피 장치
- 내부 보고 전환 가능성 상승
- 운영 부담 증가 없음

**실:**
- 파일 유통 통제 불가
- 맥락 절단 위험
- 강도 조기 노출 가능성

**조건부 채택 기준:**
- 섹션 단위 인쇄만 허용
- 상단 버튼 금지 (중·하단 배치)
- 인쇄 전용 CSS로 강도·상세 섹션 숨김
- 푸터 워터마크 고정

→ **조건 충족 시 득 > 실**

#### 현재 상태

**도입 여부:** 보류

**설계 훅:** 인쇄 가능 섹션·버튼 자리 **예약**

→ 결정 즉시 무수정 또는 최소 수정으로 활성화 가능

### 6. 프로젝트 전제 (운영 현실)

**로잉프로 구조:**
- 실질적 1인 운영 구조
- 대표/담당자는 책상 근무자가 아닌 **현장·이동 중심** 동선

**홈페이지 역할:**
- 자료 제공이 아니라 **'자료를 요청해도 되는 상대라는 확신'**을 주는 것
- 자료 제공처 ❌
- 신뢰 형성 ⭕

**운영 판단 사항:**
- 브로셔·회사소개서·PDF 발송 여부는 **운영 판단**
- 웹이 이를 대체하거나 강제하지 않음

---

### 7. 섹션 구성 합의 (보류 포함)

**합의된 섹션:**
- 랜딩페이지 **섹션 구성은 존재 자체에 동의**
- **'왜 조정인가' 섹션은 필수**
  - 말 설명이 아니라 **구조·장면·배치로 드러나야 함**
  - '구조'라는 용어·표현 방식은 재검토 대상

**보류:**
- '상황 요약'이라는 명칭·내용은 **재정의 필요 → 보류**
- 문장 길이, 설명 깊이, 읽기 리듬은 **결정 사안 아님 → 통과**
