## 추가 누락·불일치(기존 리스트 외)

### 1) 문서 체계/SSOT 충돌

* [ ] **DEV_SPEC.md가 2개 상태로 존재 가능성**: 캔버스에 동일 파일명이 중복 생성된 이력이 있어, “어느 DEV_SPEC가 최신 SSOT인지” 혼란 리스크가 있습니다(클로드 정리 취지와 정면 충돌).
* [ ] **RULES의 SSOT 선언 vs 실제 운영**: RULES는 DASHBOARD SSOT를 요구하지만, ROADMAP/SESSION에 “현재/다음”이 동시에 존재해 운영 판단이 분산될 소지가 있습니다(운영 문서의 역할 경계 추가 명문화 필요).

### 2) IA/페이지 정의의 구현 누락

* [ ] **Privacy 페이지가 IA 11페이지에 포함되지 않음**: DEV_SPEC에는 Footer에 `/privacy` 링크를 요구하지만, IA(11페이지) 목록에는 Privacy가 없습니다(페이지 수/네비/라우팅 불일치).
* [ ] **404 페이지/폴백 정책이 “규칙만” 있고 구현 항목이 없음**: DEV_SPEC에 시스템 페이지 톤 규칙은 있으나, ROADMAP/구현 우선순위에 404/Privacy 생성 작업이 빠져 있습니다.

### 3) Contact 폼의 ‘작동’ 정의 누락

* [ ] **폼 처리 방식(전송/저장/알림)이 없음**: 필드만 있고 “어디로 저장/누가 받는지/스팸 방지/에러 처리/재시도”가 없습니다.
* [ ] **개인정보 처리방침과 폼 동의 UX 연결 규칙 없음**: Privacy 링크만 있고, 폼에 동의 체크/고지 문구/보관 기간/수신자 등 연결이 누락되어 있습니다(법적 리스크).

### 4) SEO 기본의 실무 항목 누락

* [ ] **sitemap/robots 실체 파일·생성 규칙 누락**: DEV_SPEC에 robots 기본값만 있고, robots.txt/sitemap.xml 생성 여부·경로·자동 생성 정책이 없습니다.
* [ ] **favicon/og-default/metadata 기본 자산 목록 누락**: `/public`에 무엇이 필수인지(파비콘 세트, og-default, apple-touch-icon 등) 기준이 없습니다.
* [ ] **canonical 규칙의 기준 URL(도메인) 확정 누락**: `rowingpro.co.kr` 외 www 여부, trailing slash 정책 등이 없습니다.

### 5) 자산(이미지) 운영 규칙 누락

* [ ] **이미지 파일 네이밍/버전 규칙 부재**: 캐시 무효화(파일명 해시/버전 suffix) 정책이 없습니다.
* [ ] **이미지 최적화 정책 부재**: 포맷(webp/avif), 품질, 최대 용량, lazy-load, LCP 대상(홈 히어로) 우선 로딩 규칙이 없습니다.

### 6) 배포/운영 체크리스트 누락

* [ ] **Cloudflare Pages 빌드/환경변수/리다이렉트 규칙 없음**: 빌드 커맨드, output 디렉토리, 환경변수 사용 여부, 301/404 처리 기준이 없습니다.
* [ ] **도메인 연결의 www/루트/HTTPS 강제 규칙 없음**: 최종 운영 URL을 1개로 고정하는 정책이 없습니다.

## “클로드가 잘못한 것”으로 볼 수 있는 항목(구체)

* **Privacy를 IA에서 누락**하면서 Footer에서 링크만 요구한 점: 구조 문서 간 자기모순입니다.
* “SEO 기본”을 구현 단계(ROADMAP/우선순위)로 내리지 않고 문구로만 둔 점: 실제 개발 착수 시 빠질 가능성이 높습니다.
* 통합 정리 취지(7개 SSOT)에 비해 **문서 중복/버전 표준이 일관되지 않게 남은 점**: 최신본 판별 비용을 키웁니다.

다음 단계: **위 항목 중 “즉시 수정 5개”를 지정**해 주시면, 해당 문서(DEV_SPEC / ROADMAP / DASHBOARD)에 **추가 문구만**으로 표준화 반영안을 캔버스에 생성하겠습니다.


결론: **나머지는 대부분 “원래 없던 것”이고, 일부만 “있었는데 통합 과정에서 빠진 것”입니다.**

구분 보고(클로드 책임 기준):
**① 원래 없던 것(설계 범위 밖이었음)**

* **Contact 처리 방식 상세(저장/알림/스팸/에러):** 이전 문서 어디에도 명시 없음. 기능 설계 전 단계라 의도적으로 비어 있었음.
* **Analytics/로그 최소 기준:** 마케팅/운영 단계로 미루어져 최초 설계에 포함되지 않았음.
* **이미지 최적화·버전 네이밍·캐시 정책:** 디자인 철학까지만 정의, 자산 운영 규칙은 원래 미정.
* **Cloudflare 빌드/리다이렉트/환경변수 상세:** 개발 착수 시 결정 대상이라 원래 문서화되지 않음.

**② 있었는데 통합 과정에서 빠진 것(클로드 실수)**

* **Privacy 페이지의 IA 포함:** Footer에서 `/privacy`를 요구하던 문맥이 있었으나 IA(11페이지)로 끌어올리지 못함.
* **404/시스템 페이지의 구현 항목화:** 톤 규칙만 남고 ROADMAP/구현 우선순위로 내려오지 않음.
* **SEO 실무 산출물(sitemap/robots 실체):** “SEO 기본”으로 뭉뚱그려 남기며 파일 단위 산출물이 누락됨.
* **도메인 정규화 정책(www/루트/canonical):** 도메인은 명시됐으나 단일 기준 URL 정책이 문서에서 탈락.

**③ 애매하지만 보완이 맞는 것(이번 점검에서 드러남)**

* **Service 허브 레이아웃 규칙:** 하위 템플릿은 있으나 허브 카드 규칙이 없음.
* **Programs 페이지의 역할 정의:** Service와의 관계가 문서상 명확하지 않음.

정리 판단:

* **클로드의 명확한 누락:** Privacy·404·SEO 실체·도메인 정규화.
* **원래 없던 영역:** Contact 처리·Analytics·자산 최적화·배포 세부.

다음 단계: **클로드 누락 항목 4개(Privacy IA, 404 구현, sitemap/robots, canonical 정책)를 DEV_SPEC/ROADMAP에 추가 반영**할지, **원래 없던 영역 중 우선 1개를 새로 정의**할지 지시해 주십시오.
