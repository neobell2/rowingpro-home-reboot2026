  1. 로잉프로는 B2B 교육회사 (시각적 화려함보다 신뢰성)
  2. 현재 팀(Claude + CEO)으로 충분
  3. 오버스펙 기능 추가 시 비용-효율 낮음

## 2. 당신이 검토한 오버스펙 요소들

### 검토 대상

| 요소 | 예시 | 필요? | 판단근거 |

| 역할 | 내용 | 판단 기준 |

- [ ] 모바일 breakpoint (768px, 1024px 등)
- [ ] SEO 메타 태그
- [ ] 빌드 최적화

## 5. 프로토타입 반복 프로세스

### 워크플로우

```
(1) WebGPT 답변
    ↓
(2) Design Tokens 확정
    ├─ 색상 (로고 기반)
    ├─ 폰트 (한글 선택)
    ├─ Spacing (기본 단위)
    └─ Breakpoint (반응형)
    ↓
(3) Astro 프로토타입 개발 (Claude)
    ├─ 헤더/푸터
    ├─ Home 페이지
    ├─ Service 기업·조직 카드 구조
    └─ 색상/폰트 적용
    ↓
(4) 브라우저 검수 (당신)
    ├─ localhost:3000 열기
    ├─ 색상 확인 ("이 파랑이 맞아?")
    ├─ 폰트 확인 ("따뜻한 느낌인가?")
    ├─ 레이아웃 확인 ("여백이 적당한가?")
    └─ 수정 지시 ("이 부분 수정해", "더 크게", "색 바꿔")
    ↓
(5) 수정 및 재검수 (Claude → 당신)
    ↓
(6) 반복 (3~5번)
    ↓
(7) 최종 확정
```

### 당신의 피드백 예시

**좋은 피드백:**
- "이 파랑이 로고 파랑이랑 다른데?"
- "폰트가 차갑게 느껴져. 좀 더 따뜻한 느낌으로"
- "여백이 너무 많아. 조금 줄여줘"
- "모바일에서 텍스트가 너무 작아"

**피하는 피드백:**
- "CSS Flexbox를 다르게 해줘" (당신 권한이 아님)
- "타이포그래픽 스케일을 4:6:8:12로 해줘" (기술 결정)

## 6. Design Tokens 확정 단계 (다음)

### WebGPT 답변 후 당신의 결정사항

**1. 색상 (Color Palette)**
- 기준: 로고 컬러 + 설계 단계 참고
- 당신이 선택:
  - Primary Color (주 색상)
  - Secondary Color (보조 색상)
  - Neutral Colors (흰색, 회색 톤)
  - Accent Color (강조 색상)

**2. 폰트 (Typography)**
- 기준: 한글 서체 (Noto Sans, Pretendard, Spoqa Han Sans 등)
- 당신이 선택:
  - 제목 폰트
  - 본문 폰트
  - 폰트 크기 기준 (대/중/소)

**3. Spacing (여백)**
- 기준: 8px 단위 권장
- Claude 제안 후 당신 확인:
  - 기본 여백 크기
  - 요소 간 간격

**4. Responsive Breakpoint**
- 기준: Mobile (768px 이하), Tablet (768~1024px), Desktop (1024px 이상)
- Claude 설정, 당신은 최종 모습만 검수

## 8. 결론 및 원칙

### 이 프로젝트의 시각 철학

✅ **DO:**
- 신뢰성, 안정성, 통제된 에너지
- 기본에 충실 (레이아웃, 색상, 폰트)
- 모든 기기에서 완벽한 반응형
- 당신의 최종 결정이 모든 시각 선택의 중심

❌ **DON'T:**
- 화려함, 복잡한 애니메이션, 3D 효과
- "멋있어 보이려고" 하는 기능들
- 모바일에서 깨지는 시각 효과
- Claude가 기술적으로 판단하는 디자인

### 당신의 권한

> **당신이 "안 맞아"라고 하면 그게 맞지 않는 것이다.**
> Claude는 기술 구현만 하고, 최종 시각 판단은 당신이다.
