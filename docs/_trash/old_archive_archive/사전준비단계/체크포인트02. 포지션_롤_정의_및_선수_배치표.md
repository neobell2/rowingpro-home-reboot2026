# 웹사이트 리부트 프로젝트 – 포지션·롤 정의 및 선수 배치

## 0. 목적
- 작업 효율 저하의 가장 큰 원인인 **역할 중복·권한 충돌 방지**
- 각 AI/도구를 ‘능력’이 아니라 **포지션 기준으로 고정**
- 이후 모든 작업 지시는 본 표를 **전제 조건**으로 삼음

---

## 1. 포지션 롤 정의 (What, Why)

> 기준: **웹디자인 프로덕션 + 개발팀 표준 프로젝트 구성**을 따름
> (PM / 문서관리 / 디자인 / 개발 / QA 분리)

### 1) 프로젝트 매니저 (PM)
- **역할**: 일정·우선순위·의사결정 흐름 관리
- **권한**: 작업 순서 결정, 중단/재개 판단
- **금지**: 디자인·구현 직접 개입

---

### 2) 문서·컨텍스트 관리자
- **역할**: 기준 문서, 결정 사항, 변경 이력 관리
- **권한**: 문서 갱신·버전 고정
- **금지**: 판단 개입

---

### 3) 총괄 디렉터 / 콘셉트 오너
- **역할**: 미학 기준, 브랜드 방향, 최종 판단
- **권한**: Yes/No 결정권
- **금지**: 세부 구현 집착

---

### 4) 기획·정보구조 설계자
- **역할**: IA, 페이지 구조, 사용자 흐름
- **권한**: 구조 변경 제안
- **금지**: 시각 과잉 제안

---

### 5) 시각 디자인 / 비주얼 디렉션
- **역할**: 이미지, 색, 톤, 레이아웃 감각
- **권한**: 비주얼 채택/폐기 제안
- **금지**: 기술 연출 주도

---

### 6) 인터랙션·3D 설계자
- **역할**: 3D 오브젝트, 반응형, 스크롤 리듬
- **권한**: 연출 필요 여부 판단
- **금지**: 기술 과시

---

### 7) 개발 실행 엔진
- **역할**: 코드 작성, 구현, 파일 생성
- **권한**: 구현 방식 선택
- **금지**: 기획·미학 판단

---

### 8) QA / 검수·브레이크
- **역할**: 논리 오류, 과잉, 위험 요소 차단
- **권한**: 문제 제기
- **금지**: 창의 주도

---

### 9) 대안·보조 생성기
- **역할**: 변주안, 비교안 제공
- **권한**: 제안만 가능
- **금지**: 단독 결정


---


### 2) 기획·구조 설계자
- **역할**: 정보 구조, 페이지 논리, 사용자 흐름 설계
- **권한**: 구조 변경 제안
- **금지**: 시각 과잉 제안, 구현 디테일 개입

---

### 3) 시각 콘셉트·이미지 디자이너
- **역할**: 히어로 이미지, 섹션 비주얼, 시각 톤 정의
- **권한**: 이미지 채택/폐기 제안
- **금지**: 브랜드 철학을 벗어난 스타일 실험

---

### 4) 인터랙션·3D 연출 설계자
- **역할**: 3D 오브젝트, 반응형 아트, 스크롤·리듬 설계
- **권한**: 연출 필요 여부 판단
- **금지**: 기술 과시, 메시지 분산

---

### 5) 실행 엔진 (빌더)
- **역할**: 코드 작성, 파일 생성, 실제 구현
- **권한**: 구현 방식 선택
- **금지**: 기획·미학 판단

---

### 6) 검수·브레이크
- **역할**: 논리 오류, 과잉, 위험 요소 차단
- **권한**: 문제 제기 권한 (결정권 아님)
- **금지**: 창의 주도

---

### 7) 대안·보조 생성기
- **역할**: 다른 선택지, 변주안 제공
- **권한**: 제안만 가능
- **금지**: 단독 결정

---

## 2. 포지션 × 선수 배치 표 (웹프로덕션 기준 재정렬)

| 포지션 | 웹프로덕션 기준 역할 | 배정 선수 | 비고 |
|---|---|---|---|
| PM | 일정·우선순위·흐름 | 사용자(본인) | 최종 통제 |
| 문서·컨텍스트 | 기준/결정 관리 | ChatGPT | 단일 소스 |
| 총괄 디렉터 | 미학·브랜드 | 사용자 + ChatGPT | 사용자 최종 |
| IA·기획 | 구조·흐름 | ChatGPT | 주전 |
| 비주얼 디자인 | 이미지·톤 | ChatGPT(주력) / Gemini(보조) | Gemini 단독 금지 |
| 인터랙션·3D | 리듬·연출 | ChatGPT(설계) + Roo/Codex(구현) | 절제 원칙 |
| 개발 실행 | 코드·빌드 | Roo + Codex | 동일 엔진 |
| QA/검수 | 안정성 | Cursor + Claude | 전용 |
| 보조 생성 | 대안 | Gemini 3.0 | 선택권 없음 |
| 코드 참고 | 스니펫 | DeepSeek | 디자인 배제 |

---|---|---|---|
| 총괄 디렉터 | 방향·판단·미학 기준 | 사용자(본인) + ChatGPT | 판단은 사용자 최종 |
| 기획·구조 설계 | 정보 구조·흐름 | ChatGPT(웹, 고급) | 주전 |
| 시각 콘셉트·이미지 | 이미지·톤 | ChatGPT(주력) / Gemini 3.0(보조) | Gemini 단독 금지 |
| 인터랙션·3D | 3D·반응형 연출 | ChatGPT(설계) + Roo/Codex(구현) | 리듬 중심 |
| 실행 엔진 | 코드·파일·구현 | Roo + Codex (VSCode / Antigravity) | 동일 엔진 |
| 검수·브레이크 | 보수적 검토 | Cursor + Claude | 전용 포지션 |
| 대안·보조 | 변주·비교안 | Gemini 3.0 | 선택권 없음 |
| 코드 보조 | 스니펫·참고 | DeepSeek | 디자인 영역 배제 |

---

## 3. 핵심 운용 원칙

- 한 포지션에 **결정권은 하나**
- 검수 포지션은 **항상 늦게 개입**
- 실행 엔진은 **판단하지 않는다**
- 이미지·연출은 **‘통제된 에너지’ 원칙 고수**

---

## 4. 상태
- 이 문서는 **롤 충돌 방지용 기준 문서**
- 이후 모든 작업 지시는
  → `포지션 + 대상 + 산출물` 형태로 명시

예) “시각 콘셉트 포지션에서 히어로 이미지 2안 생성”
