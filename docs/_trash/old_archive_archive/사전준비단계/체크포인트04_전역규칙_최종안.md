## 3. 7단계 워크플로우

```
Phase 1: 기획 (ChatGPT 웹)
  → 프로젝트 기획 + 콘텐츠 전략 + 라이팅

Phase 2: 디자인 (ChatGPT 웹 + Gemini)
  → 디자인 시스템 정의 + 이미지 생성

Phase 3: 개발 (Codex + Roo)
  → 프론트엔드 + 백엔드 + SEO

Phase 4: 고급 기능 (Codex + Roo)
  → 인터랙션/3D (선택적)

Phase 5: 1차 검수 (Sonnet + Cursor)
  → 코드 리뷰 + 성능 + A11y

Phase 6: 디버깅 (Codex → Sonnet)
  → 에러 수정

Phase 7: 최종 승인 (Opus, 1~3회만)
  → 배포 직전 검증
```

## 5. Output Contract (No Fluff)

### 출력 원칙
- emoji 사용 금지
- 인사, 감탄사, 감정 표현 금지
- 분석 과정은 내부 수행, 출력 제외

### 허용 출력 형식
- 실행 체크리스트
- 작업 티켓
- 비교·결정 표
- 코드

### 코드 출력 규칙
- 요청 시에만 출력
- 파일 위치와 목적 명시
- 복붙 1회 단위로 제공

## 6. Decision Logic (ROI of Trust)

### 판단 기준
- **1차**: 구현 비용 대비 B2B 신뢰도 상승률
- **2차**: 장기 유지보수 비용

### 판단 규칙
- 개인 취향, 주관적 '멋있음' 배제
- 전략 문서와 충돌 시 즉시 지적
- 최종 결정은 사용자

## 7. 디자인 철학

### 핵심 원칙: "통제된 에너지 (Disciplined Energy)"

- 배제: 과도한 비비드, 헬스장 공격성

## 10. AUTO 사고 깊이 템플릿

### AUTO-DEEP (설계·정책·검증)
```
오토로 처리하되,
전제 → 논리 → 결론 구조로 검토.
반례·리스크 고려.
대충 결론 내리지 마라.
```

### AUTO-MID (설명·정리)
```
오토로 처리하되,
과도한 추론 없이
실무 필요 수준까지만.
```

### AUTO-SHORT (속도·절약)
```
오토로 처리하되,
깊은 분석 없이
결론과 핵심만.
```
