# 웹사이트 리부트 프로젝트 – 체크포인트 (원클릭)

## 0. 목적
- 기존 엔터프라이즈급 프로젝트와 **완전히 다른 성격의 작업**으로 전환
- **디자인·표현 중심**의 소규모 웹사이트 구축
- AI를 주력으로 활용하는 **새 워크플로우 확정**

---

## 1. 현재 단계
- **선수 배치 + 작전 수립 단계**
- 실행 이전의 **전제·역할·원칙 고정**이 목표

---

## 2. 인력(모델) 배치 – 확정

### 주전
- **ChatGPT (웹, 고급 모델)**
  - 기획 정리
  - 정보 구조
  - 시각 콘셉트 해석
  - 이미지 주력 생성

### 실행 엔진
- **Roo + Codex** (동일 본질)
  - 환경: VS Code / Antigravity
  - 역할: 구현·생산·파일 작업
  - 차이: 운용 옵션(무료 토큰, 멀티 에이전트 등)

### 검수관
- **Cursor + Claude (월 구독, 전용)**
  - 보수적 검토
  - 과잉 생성 차단
  - 논리·안전성 검수

### 보조
- **Gemini 3.0**
  - 대안 이미지/문구 변주
  - 현장감·다변화 보강
- **DeepSeek**
  - 코드 스니펫 보조 한정
  - 이미지/미학 영역에서는 배제

---

## 3. 도구 세팅 – 공통 이해
- Cursor / VS Code / Antigravity에 **Codex 모두 탑재**
- 능력 차이 없음
- 차이는 **운용 방식과 자원 조건**뿐
- 도구 설명 단계 종료 → **작전과 지시의 질이 관건**

---

## 4. 디자인·미학 핵심 원칙

### 기본 방향
- 스포츠 회사로서 **역동성 필수**
- 그러나 과잉 금지

### 배제
- 영패션/스트리트식 과도한 비비드
- 헬스·크로스핏식 공격성

### 지향
- **조정(Rowing) 종목 특성 반영**
  - 동기화
  - 수평 이동
  - 지속되는 집중과 리듬
- 키워드: **통제된 에너지 (Disciplined Energy)**

---

## 5. 타깃 전략
- **B2B + 일반 고객 동시 어필**
  - B2B: 통제력, 신뢰, 안정성
  - 일반: 멋있음, 역동성, 접근성
- 동일한 시각 요소를 **다르게 읽히게 만드는 구조**

---

## 6. 이미지·그래픽 전략
- 목적: 눈에 띄는 그림 ❌ / 텍스트를 방해하지 않는 그림 ⭕
- 기준:
  - 맥락 이해
  - 레이아웃·문구와의 정합성
- 주력 생성: **ChatGPT**
- 대안 생성: **Gemini 3.0**

---

## 7. 3D 오브젝트 · 반응형 아트 기준

### 필요성
- 있음 (조건부)

### 금지
- 기술 과시
- 메시지 분산
- 과도한 인터랙션

### 허용
- 노의 반복 궤적
- 물결·리듬·패턴
- 스크롤/마우스에 반응하는 **리듬 중심 요소**

### 원칙
- **오브젝트는 말하지 않는다**
- **리듬만 보여준다**
