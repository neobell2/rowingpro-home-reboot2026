# 로잉프로 홈페이지 리부트 - 문서 운용 규칙

**버전:** v1.0 Final
**일자:** 2026. 01. 29.
**목적:** 웹 ChatGPT와 로컬 도구 간 협업 체계 확립

---

## 1. 기본 전제

1. **로컬 도구**(Cursor, Roo)는 같은 Git 저장소 공유 → 자동 동기화
2. **웹 ChatGPT**만 격리됨 (수동 복붙 필요)
3. Git = 단일 진실 원천 (버전 관리 + 배포 연결)
4. 4일 프로젝트 기준, 최소 문서만 관리

---

## 2. 역할 구분

### ChatGPT (웹)
- 기획, 콘텐츠, 이미지 생성
- 중요 결정 판단
- Git 파일을 수동으로 받아서 검토

### Cursor + Roo (로컬)
- 모든 개발 작업
- Git 직접 관리
- 파일 자동 동기화

---

## 3. 문서 3계층

### Tier 1: 기준 문서 (프로젝트 시작 시 1회)
- `README.md` - 프로젝트 개요, 기술 스택
- `RULES.md` - 전역 규칙
- `TEAM.md` - AI 배치 및 역할

### Tier 2: 작업 문서 (진행 중 수시)
- `SESSION.md` - 현재 세션 상태
- `PROGRESS.md` - 진행 현황
- 기획 문서들

### Tier 3: 결과 문서 (완료 후 1회)
- `HANDOFF.md` - 배포 후 유지보수 가이드

---

## 4. 운영 원칙 (MLO 참고)

### Stop-Scan-Snipe (문제 발생 시)
1. **Stop**: 즉시 작업 중단
2. **Scan**: 로그/코드 증거 확보
3. **Snipe**: 정밀 수정 (추측 금지)

### 선택적 보고
- **P0 (치명적)**: 즉시 보고 필수
- **P1 (중요)**: 즉시 보고 필수
- **P2/P3**: 요청 시에만 보고

---

## 5. 웹 ↔ 로컬 전달 규칙

### 업로드 (로컬 → 웹)
1. Cursor가 `HANDOFF.md` 생성
2. 사용자가 ChatGPT Projects에 복붙
3. 중요 결정 필요 시에만

### 다운로드 (웹 → 로컬)
1. ChatGPT 결정 사항 복사
2. Cursor가 파일 업데이트
3. **즉시 Git 커밋** (필수)

### 원칙
- 자동화 시도 안 함
- 모든 작업은 Git에 기록
- **웹 결정 → 즉시 반영 → 즉시 커밋**

---

## 6. 세션 관리 프로토콜 (휴먼에러 방어)

### 목적
맥락 손실 방지 (중단 시 복구)

### SESSION.md 포맷

**작업 시작 시:**
```markdown
## 세션: 2026-01-29 14:30
- 현재 작업: [작업명]
- 사용 도구: ChatGPT / Cursor / Roo
- 결과물: [파일명]
- 다음 단계: [다음 할 일]
```

**중단 시 (필수):**
```markdown
## 중단: 2026-01-29 15:45
- 완료: [완료한 것]
- 미완료: [안 끝난 것]
- 복붙 상태: ✅ 완료 / ❌ 미완료
```

**재개 시:**
1. SESSION.md 읽기
2. 미완료 항목부터 처리
3. Git 로그 확인

### 규칙
- 5분 이상 자리 비우면 업데이트
- Git 커밋 전 SESSION.md 확인
- 모든 세션 종료 시 상태 기록
