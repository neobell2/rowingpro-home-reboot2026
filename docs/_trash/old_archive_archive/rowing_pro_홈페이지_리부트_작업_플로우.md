# 홈페이지 리부트 작업 플로우 정리

본 문서는 RowingPro 홈페이지 전면 리부트를 **내부 인력(1인 작업)** 기준으로 실행하기 위한 실제 작업 흐름을 정리한 문서입니다.  
외부 웹빌더나 외주가 아닌, **로컬 IDE 기반 직접 제작**을 전제로 합니다.

---

## 1. 기본 전제

- 외주 제작 ❌
- 웹빌더(UI 기반 서비스) ❌
- 로컬 IDE에서 직접 작업 ⭕
- 무료 또는 이미 보유한 도구만 사용

### 사용 환경
- VSCode
- Roo (IDE 내 코딩 에이전트)
- Gemini (기획·문서·설명)
- DeepSeek (코드 생성)
- Astro (정적 웹사이트 생성)
- Cloudflare Pages (무료 배포)
- Google Workspace (메일·문서·문의)

---

## 2. 전체 작업 흐름 개요

```
[기획/문구 정리]
        ↓
[정보 구조 설계]
        ↓
[로컬 환경 구축]
        ↓
[페이지/컴포넌트 구현]
        ↓
[디자인 조정]
        ↓
[배포 및 전환]
```

각 단계는 **서로 겹치지 않도록 순차 진행**하는 것이 핵심입니다.

---

## 3. 단계별 상세 작업 플로우

---

### STEP 1. 기획 및 문구 정리

**목적**  
홈페이지에서 전달해야 할 메시지를 명확히 정의

**주요 작업**
- 회사 한 줄 소개
- 대표 메시지
- 주요 서비스/프로그램 설명
- 공식 홈페이지 톤 정리

**사용 도구 / 모델**
- Gemini

**산출물**
- `content.md`
- `about.md`

※ 이 단계에서는 코딩을 시작하지 않음

---

### STEP 2. 정보 구조 설계

**목적**  
홈페이지 전체 구조를 단순하고 명확하게 고정

**주요 작업**
- 페이지 수 확정 (4~5개 권장)
- 각 페이지별 섹션 정의

**권장 페이지 구성 예시**
- Home
- About
- Programs / Services
- Track Record
- Contact

**사용 도구 / 모델**
- Gemini

**산출물**
- 사이트맵
- 페이지별 섹션 목록

※ 이 단계 이후 구조 변경은 최소화

---

### STEP 3. 로컬 개발 환경 구축

**목적**  
실제 제작을 위한 기술 환경 준비

**주요 작업**
- Astro 프로젝트 생성
- Tailwind CSS 설정
- 기본 레이아웃 구성

**사용 도구 / 모델**
- VSCode
- Roo
- DeepSeek

**결과**
- 로컬에서 실행 가능한 기본 사이트

---

### STEP 4. 페이지 및 컴포넌트 구현

**목적**  
실제 화면 구성

**주요 작업**
- 페이지 파일 생성
- 공통 레이아웃 컴포넌트화
- 텍스트 우선 배치 (이미지는 임시)

**사용 도구 / 모델**
- Roo (파일 생성 및 수정)
- DeepSeek (Astro / Tailwind 코드)

**주의 사항**
- 이 단계에서는 디자인 완성도보다 구조 우선

---

### STEP 5. 디자인 및 가독성 조정

**목적**  
비전문가 기준으로도 읽기 쉬운 화면 구성

**주요 작업**
- 여백 조정
- 폰트 크기 및 계층 정리
- 모바일 반응형 점검

**사용 도구 / 모델**
- Gemini (어색한 지점 판단)
- Roo (코드 수정)

---

### STEP 6. 이미지 반영

**목적**  
신뢰도 확보

**주요 작업**
- 실사진 위주 반영
- 히어로 이미지 1장 (필요 시 AI 이미지 사용)

**주의 사항**
- 이미지 수 최소화
- 과도한 AI 이미지 사용 지양

---

### STEP 7. 배포 및 전환

**목적**  
정상적인 공식 홈페이지 공개

**주요 작업**
- GitHub 저장소 연결
- Cloudflare Pages 배포
- HTTPS 자동 적용 확인
- 도메인 연결

**사용 도구 / 서비스**
- Cloudflare Pages

**결과**
- 보안 경고 없는 공식 홈페이지 운영 시작

---

## 4. 역할 분담 요약

| 역할 | 담당 |
|---|---|
| 기획·문구·설명 | Gemini |
| 구조 설계 | Gemini |
| 코드 생성 | DeepSeek |
| 파일 관리/수정 | Roo |
| 최종 판단 | 작업자 |

---

## 5. 작업 원칙 (중요)

- 한 번에 완벽을 목표로 하지 않음
- 구조를 먼저 고정하고 디테일은 나중에 조정
- 웹사이트는 “운영 인프라”로 접근
- 외부 의존도 최소화

---

## 6. 요약

본 리부트 작업은:
- 비용을 최소화하면서
- 내부 통제력을 확보하고
- 안정적인 공식 홈페이지를 빠르게 구축하는 것이 목적입니다.

**2~3일 내 1차 완성 → 이후 점진적 개선**을 기준으로 진행합니다.
