# 08. 구현 계획 (Phase 3)

Astro와 Tailwind CSS를 사용하여 로잉프로 홈페이지의 핵심 구조를 초기화하고 구축합니다.

## 핵심 사항
- **프레임워크**: Astro (고성능, 무료 호스팅 최적화)
- **스타일링**: Tailwind CSS (브랜드 컬러 적용)
- **페이지 구성**: 총 11개 페이지 (Home, About, Programs, Service Hub, Track Record, Contact + 서비스 하위 5개)

## 콘텐츠 주입 및 우선순위 (Workflow)

사용자 피드백과 `DEV_CONTENT_SPEC.md`에 따라, 단순 placeholder 대신 실시간으로 콘텐츠를 주입하며 개발합니다.

1. **초기 구축 단계**: `Layout.astro` 구조 확정 시 메타 태그 및 OG 문구를 즉시 고정합니다.
2. **레이아웃/컴포넌트 단계**: Home, About, Programs, Track Record 컴포넌트 구현과 동시에 브랜드 메시지를 바탕으로 한 텍스트를 주입합니다.
3. **서비스 템플릿 단계**: 공통 템플릿(옵션 A) 구현 후 5개 서브 페이지의 특화된 텍스트 블록을 즉시 채워 넣습니다.
4. **전환/필터 단계**: FAQ 문안을 서비스 공통 섹션과 Contact 페이지의 보조 필터(질문 유도) 용도로 선행 배치합니다.

## 상세 계획

### 1. 기반 및 레이아웃
- Astro 프로젝트 초기화
- 전역 Tailwind 설정 (워터 블루, 다이내믹 오렌지 등)
- 공통 레이아웃 (`Layout.astro`), 헤더, 푸터 구현
- 서비스 드롭다운 메뉴가 포함된 네비게이션 구현

### 2. 주요 컴포넌트
- **ServiceTemplate**: 서비스 하위 5개 페이지용 공통 템플릿 (옵션 A)
- **ImageSlider**: 16:9 비율의 반응형 이미지 슬라이더
- **ContactForm**: `source_page` 자동 전달 로직이 포함된 문의 폼

### 3. 페이지 생성
- `index.astro` (홈)
- `about.astro` (소개)
- `programs.astro` (프로그램)
- `service/index.astro` (서비스 허브)
- `track-record.astro` (실적)
- `contact.astro` (문의)
- `service/corporate.astro` (기업/조직)
- `service/school-youth.astro` (학교/청소년)
- `service/event.astro` (이벤트/행사)
- `service/club-general.astro` (클럽/일반)
- `service/family.astro` (가족체험)

## 검증 방법
- `npm run build` 성공 여부 확인
- GNB 호버 및 모바일/데스크톱 반응형 동작 수동 확인
- 모든 페이지 URL 접근성 확인
