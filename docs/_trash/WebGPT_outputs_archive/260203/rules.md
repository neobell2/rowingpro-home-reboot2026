## 1. 권한

| 역할     | 권한                   |
| ------ | -------------------- |
| 사용자    | 최종 결정권자              |
| 담당 AI  | 설계/판단/실행 (사용자 승인 하에) |
| WebGPT | 초안만 (검증 필수)          |

---

## 2. 핵심 원칙

- 작업 1개 = 세션 1개
- 파일 1개씩 처리
- 전체 프로젝트 스캔 금지
- 추측 금지, 모르면 질문

---

## 3. 출력 규칙

**금지:** emoji, 인사, 감탄사, 불필요한 설명 **허용:** 실행 결과, 에러, 다음 단계

---

## 4. 파일 작업 규칙

- 생성/수정 전 경로 확인
- 전체 재작성 금지 (지시 없는 한)
- 변경 범위 최소화
- 완료 후 다음 파일로 이동

---

## 5. WebGPT 산출물 처리

```
WebGPT 초안 → 사용자 선별 → AI 검증 → 적용
```

- 1000행 이하: OK
- 1000행 초과: 요약 후 투입

---

## 6. 문서 관리 원칙

| 문서           | 역할            | 업데이트 시점 |
| ------------ | ------------- | ------- |
| DASHBOARD.md | 단일 진실원천(SSOT) | 매 세션    |
| SESSION.md   | 작업 이력         | 매 세션    |
| ROADMAP.md   | 마일스톤 관리       | 단계 완료 시 |
| README.md    | 고정 개요         | 변경 시    |

---

## 7. 문서 우선순위 (운영)

문서 간 충돌 시 다음 우선순위를 따른다.

1. 최신 작업지시서
2. **DASHBOARD.md** (현재 상태·블로커·다음 작업, SSOT)
3. SESSION.md (이력)
4. README.md (고정 개요)

---

## 8. 세션 프로토콜

### 시작

1. DASHBOARD.md 읽기
2. 역할/작업 확인
3. 작업 시작

### 종료 (SSOT 준수)

1. 완료 항목을 SESSION.md에 기록
2. **DASHBOARD.md를 단일 진실원천(SSOT)으로 업데이트**
3. 다음 작업 명시

> 모든 진행 판단과 현재 상태 확인은 **DASHBOARD.md 기준**으로 한다.
