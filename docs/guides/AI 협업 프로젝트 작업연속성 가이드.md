# AI 협업 프로젝트 작업연속성 가이드

## 목적

계정 변경, 세션 종료, AI 교체 상황에서도 **설계·코드 방향이 흔들리지 않도록** 프로젝트 맥락을 문서로 고정한다.

---

# 필수 문서 (5개)

| 문서                 | 역할         | 핵심 내용                  |
| ------------------ | ---------- | ---------------------- |
| 01_PROJECT_CONTEXT.md | 프로젝트 전체 정의 | 목적, 범위, 금지사항, 기술스택     |
| 02_ARCHITECTURE.md    | 구조 확정본     | 사이트맵, 레이아웃, 공통 규칙      |
| 03_DECISIONS.md       | 결정 + 이유 기록 | 무엇 / 왜 / 대안 / 변경 가능 여부 |
| 04_CURRENT_STATE.md   | 현재 진행 위치   | 완료, 진행중, 이슈            |
| 05_NEXT_TASK.md       | 즉시 실행 지시서  | 다음 작업 목표와 제약           |

👉 이 5개가 존재하면 **이전 대화 없이도 새 AI가 바로 작업 가능**해야 한다.

---

# 권장 폴더 구조

```
/docs
 ├ 01_PROJECT_CONTEXT.md
 ├ 02_ARCHITECTURE.md
 ├ 03_DECISIONS.md
 ├ 04_CURRENT_STATE.md
 └ 05_NEXT_TASK.md
```

문서는 반드시 **한 폴더에 집약**한다.
여러 위치에 분산되면 AI가 맥락을 놓친다.

---

# 문서별 작성 기준

## 01_PROJECT_CONTEXT.md

프로젝트의 “절대 기준”.

반드시 포함:

* 프로젝트 목적 (예: 회사소개 웹사이트 10p)
* 현재 단계 (예: 디자인 확정 → 마크업 진행)
* 절대 변경 금지 요소

  * 컬러
  * 네비 구조
  * 브랜드 톤
* 기술 스택

👉 기준 문서이므로 **자주 수정하지 않는다.**

---

## 02_ARCHITECTURE.md

구조가 흔들리는 것을 방지하는 문서.

반드시 포함:

* 사이트맵
* 공통 레이아웃 (Header / Footer / Grid 등)
* 디자인 시스템 핵심값
* 컴포넌트 규칙

구조 변경이 발생하면 즉시 갱신한다.

구조 문서가 없으면 AI는 매번 새 구조를 제안하려 한다.

---

## 03_DECISIONS.md

가장 중요한 문서.

기록 형식은 항상 동일하게 유지:

```
결정:
이유:
대안:
변경 가능 여부:
```

예:

```
결정: 상단 고정 네비게이션 사용
이유: 기업사이트 신뢰성 확보
대안: 햄버거 메뉴 → 폐기
변경 가능 여부: 낮음
```

👉 핵심은 **“왜”**다.
이유가 없으면 동일한 논쟁이 반복된다.

---

## 04_CURRENT_STATE.md

프로젝트의 현재 좌표.

형식:

```
완료:
진행중:
이슈:
```

작업 종료 시 반드시 업데이트한다.

이 문서가 없으면 다음 세션에서 항상 다음 질문이 발생한다:

👉 “지금 어디까지 했는가?”

---

## 05_NEXT_TASK.md

AI에게 그대로 입력 가능한 작업 지시서.

형식:

```
목표:
요구사항:
금지사항:
```

예:

```
목표: About 페이지 마크업
요구사항: Home 디자인 시스템 유지, 반응형 필수
금지사항: 디자인 변경 금지
```

작업 시작 전에 작성한다.

NEXT 없이 작업하면 방향이 흔들린다.

---

# 운영 규칙 (핵심 3개)

| 규칙              | 효과       |
| --------------- | -------- |
| 결정 발생 즉시 기록     | 설계 붕괴 방지 |
| 작업 종료 시 상태 업데이트 | 연속성 확보   |
| 작업 시작 전 NEXT 작성 | 실행 속도 증가 |

세 가지만 지켜도 프로젝트 안정성이 크게 올라간다.

---

# 흔한 실패 원인

| 원인          | 결과         |
| ----------- | ---------- |
| 결정 이유 미기록   | 동일 논쟁 반복   |
| CURRENT 미작성 | 작업 공백 발생   |
| NEXT 없이 시작  | AI가 방향 재설계 |
| 문서 분산       | 맥락 손실      |

---

# 안정성 판단 기준

다음 질문에 **문서만 보고 답 가능하면 정상 구조**:

* 무엇을 만드는 프로젝트인가
* 구조는 확정되었는가
* 왜 그렇게 설계했는가
* 지금 어디까지 했는가
* 다음에 무엇을 해야 하는가

하나라도 답이 안 나오면 문서 보강 필요.

---

# 핵심 원칙

👉 **AI가 기억해야 할 것은 대화가 아니라 문서에 존재해야 한다.**

대화 기반 프로젝트는 세션이 끊기는 순간 흔들린다.
문서 기반 프로젝트는 어떤 AI로 교체해도 계속 진행된다.
